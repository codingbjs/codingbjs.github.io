통신 규격 및 절차

a. 통신 규격 및 전송 방식

  • 프로토콜 계층: TCP/IP 기반 소켓 통신을 사용하며, 포트 번호는 기본적으로 30200번을 할당합니다.
  • 바이트 오더: 다중 바이트 필드는 모두 네트워크 바이트 오더(Big-Endian)를 따릅니다.

b. 통신 절차 및 세션 관리

  • 초기 연결: VMS 장비가 서버에 TCP 연결 요청 → 서버가 인증용 Device ID 요청(0xFF) → 장비가 ID 응답.
  • 상태 확인: 인증 후 서버는 1분 주기로 상태 요구(0x05)를 전송하여 장비 상태를 모니터링합니다.
  • 세션 유지: 5분간 요청이 없으면 장비가 유효성 확인(0x12)을 보내고, 응답 없으면 3회 재시도 후 재접속합니다.
  • ACK/NAK 처리: 모든 요청에 대해 ACK(0x06) 또는 NAK(0x15+오류코드)로 응답합니다.
